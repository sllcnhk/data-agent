# 🚀 系统启动完整指南

## 第一步：确保后端正常启动

### 检查后端是否启动成功

**方法1: 直接访问后端**
```bash
# 浏览器访问或curl
http://localhost:8000/health
```

如果返回JSON，说明后端正常启动。

**方法2: 检查后端日志**
启动后端后，查看终端窗口输出，应该看到：
```
INFO: Started server process [xxxxx]
INFO: Uvicorn running on http://0.0.0.0:8000
```

---

## 第二步：配置数据库连接信息

由于您暂时没有ClickHouse和MySQL连接，暂时跳过。

后续需要配置时，编辑 `.env` 文件：
```env
# ClickHouse配置 (示例)
CLICKHOUSE_IDN_HOST=your-clickhouse-host
CLICKHOUSE_IDN_USER=your-user
CLICKHOUSE_IDN_PASSWORD=your-password

# MySQL配置 (示例)
MYSQL_PROD_HOST=your-mysql-host
MYSQL_PROD_USER=your-user
MYSQL_PROD_PASSWORD=your-password
```

## 常见启动问题解决

### Q1: 后端启动失败

**错误现象**: 终端显示错误信息后退出

**解决方法**:
1. 检查Python依赖:
   ```bash
   cd backend
   pip install -r requirements.txt
   ```

2. 检查数据库连接:
   ```bash
   python test_connection.py
   ```

3. 查看详细错误信息

### Q2: 前端无法连接后端

**错误现象**: 页面显示"连接失败"或"Network Error"

**解决方法**:
1. 确认后端在8000端口正常运行
2. 检查防火墙设置
3. 确认CORS配置正确

### Q3: MCP服务器获取失败

**错误现象**: 页面显示"获取MCP服务器列表失败"

**原因**: MCP需要实际的数据库连接配置才会显示

**解决方法**:
- 如果没有ClickHouse/MySQL，暂时忽略此错误
- 或在`.env`中配置数据库连接信息

## 完整启动流程

### 1. 启动后端
```bash
cd backend
python main.py
```

确认看到: `INFO: Uvicorn running on http://0.0.0.0:8000`

### 2. 启动前端 (新终端)
```bash
cd frontend
npm run dev
```

确认看到: `Local: http://localhost:3000`

### 3. 访问系统
- 前端: http://localhost:3000
- 后端API: http://localhost:8000
- API文档: http://localhost:8000/docs
