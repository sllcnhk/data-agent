# Data Agent 完整启动指南

## 问题总结

您的系统当前使用 **Python 3.7.0**，但项目需要 **Python 3.8+**。所有问题都源于此版本不兼容。

## 一次性完整解决方案

### 第一步：运行环境设置脚本（仅需执行一次）

在项目根目录双击运行或在命令行执行：

```cmd
setup-environment.bat
```

这个脚本会自动：
1. ✅ 创建名为 `dataagent` 的 Python 3.8 conda 环境
2. ✅ 安装所有后端依赖（兼容版本）
3. ✅ 检查并安装前端依赖
4. ✅ 完成所有必要的环境配置

**执行时间：** 约 5-10 分钟（取决于网络速度）

### 第二步：启动系统

环境设置完成后，每次启动只需运行：

```cmd
start-all.bat
```

这个脚本会自动：
1. ✅ 检测并激活 `dataagent` conda 环境
2. ✅ 验证 Python 版本（必须 3.8+）
3. ✅ 检查 Node.js
4. ✅ 检查 PostgreSQL 服务
5. ✅ 验证数据库连接
6. ✅ 启动后端服务器（http://localhost:8000）
7. ✅ 启动前端开发服务器（http://localhost:3000）

**等待时间：** 10-15 秒后访问 http://localhost:3000

## 常见问题排查

### 问题 1：setup-environment.bat 找不到 conda

**原因：** Anaconda 未添加到 PATH

**解决方案：**
1. 打开 Anaconda Prompt（从开始菜单）
2. 在 Anaconda Prompt 中运行：
   ```cmd
   cd C:\Users\shiguangping\data-agent
   setup-environment.bat
   ```

### 问题 2：start-all.bat 还是使用 Python 3.7

**原因：** 环境未正确激活

**解决方案：**
1. 手动激活环境后再运行：
   ```cmd
   conda activate dataagent
   cd C:\Users\shiguangping\data-agent
   start-all.bat
   ```

### 问题 3：端口被占用

**症状：** 提示 `Address already in use` 或端口冲突

**解决方案：**
```cmd
# 检查端口占用
netstat -ano | findstr ":3000"
netstat -ano | findstr ":8000"

# 结束占用端口的进程
taskkill /PID <进程ID> /F
```

### 问题 4：数据库连接失败

**症状：** 后端启动时提示数据库错误

**解决方案：**
1. 确认 PostgreSQL 正在运行：
   ```cmd
   sc query postgresql-x64-18
   ```
2. 如果未运行，启动服务：
   ```cmd
   net start postgresql-x64-18
   ```
3. 初始化数据库：
   ```cmd
   conda activate dataagent
   cd C:\Users\shiguangping\data-agent\backend
   python scripts\init_chat_db.py
   ```
