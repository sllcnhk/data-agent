# 手动安装步骤 - Data Agent

## ⚠️ 问题说明

自动脚本在 Windows PowerShell 中遇到 conda 环境激活问题。请按以下手动步骤操作：

---

## 🔧 手动安装步骤（推荐）

### 步骤 1：打开 Anaconda Prompt

**重要**：不要使用 PowerShell 或 CMD，必须使用 **Anaconda Prompt**

从开始菜单搜索并打开：`Anaconda Prompt`

### 步骤 2：创建 Python 3.8 环境

在 Anaconda Prompt 中运行：

```bash
conda create -n dataagent python=3.8 -y
```

等待环境创建完成（约 1-2 分钟）

### 步骤 3：激活环境

```bash
conda activate dataagent
```

### 步骤 4：验证 Python 版本

```bash
python --version
```

**必须显示**: `Python 3.8.x`（如果显示 3.7.0，说明环境未正确激活，重新执行步骤 3）

### 步骤 5：进入项目目录并安装后端依赖

```bash
cd C:\Users\shiguangping\data-agent\backend
pip install -r requirements-py38.txt
```

**预计时间**: 3-5 分钟

### 步骤 6：检查前端依赖

```bash
cd C:\Users\shiguangping\data-agent\frontend
npm install
```

如果前端依赖已经安装，会显示 "up to date"

### 步骤 7：初始化数据库（如果未初始化）

```bash
cd C:\Users\shiguangping\data-agent\backend
python scripts\init_chat_db.py
```

## ❌ 常见问题

### Q1: conda activate dataagent 后还是显示 Python 3.7

**原因**：PowerShell 中 conda 环境未正确激活

**解决**：
1. 关闭 PowerShell
2. 打开 **Anaconda Prompt**（从开始菜单）
3. 在 Anaconda Prompt 中运行 `conda activate dataagent`

### Q2: pip install 报 GBK 编码错误

**原因**：requirements 文件编码问题

**解决**：已修复，现在的 requirements-py38.txt 是纯 ASCII 编码

重新运行：
```bash
cd C:\Users\shiguangping\data-agent\backend
pip install -r requirements-py38.txt
```

### Q3: 找不到 start-all.bat

**原因**：路径不对

**解决**：
```bash
cd C:\Users\shiguangping\data-agent
dir start-all.bat
```

确认文件存在后再运行

### Q4: 端口被占用

**症状**：启动时提示 "Address already in use"

**解决**：
```bash
# 查找占用端口的进程
netstat -ano | findstr ":3000"
netstat -ano | findstr ":8000"

# 结束进程（替换 <PID> 为实际进程 ID）
taskkill /PID <PID> /F
```
