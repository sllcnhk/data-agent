# 前端日志功能使用指南

## 概述

为了方便调试和诊断"Network Error"等问题，我们添加了详细的前端日志功能。系统现在会自动记录所有API请求、响应和错误，帮助您快速定位问题根源。

## 功能特性

### 1. 自动日志记录

系统会自动记录以下信息：
- **API请求**: HTTP方法、URL、请求头、请求体
- **API响应**: 状态码、响应数据
- **错误详情**: 错误消息、堆栈跟踪、网络状态
- **时间戳**: 精确到毫秒的时间记录

### 2. 日志存储

- 日志存储在浏览器 `localStorage` 中
- 自动管理日志数量（最多1000条）
- 页面刷新后日志仍然保留

### 3. 日志查看

访问日志页面：
```
导航菜单 → 日志查看
```

或直接访问：
```
http://localhost:3000/logs
```

### 4. 日志筛选

在日志页面，您可以：
- 按级别筛选：ERROR、WARN、INFO、DEBUG
- 按类别筛选：API、NETWORK、ERROR、USER
- 按关键词搜索：搜索消息内容或详情
- 查看最新日志：按时间排序

### 5. 日志导出

点击"导出日志"按钮，将日志保存为JSON文件，方便：
- 分享给他人协助调试
- 保存调试历史记录
- 离线分析问题

## Network Error 诊断指南

当出现"Network Error"或"创建对话失败"等错误时：

### 1. 查看日志

点击错误消息中的"查看日志"按钮，或：
1. 导航菜单 → 日志查看
2. 筛选类别为"API"
3. 筛选级别为"ERROR"

### 2. 常见错误原因

根据日志中的错误详情判断：

#### 后端服务未启动
```
错误类别: NETWORK
错误消息: Network Error
详细信息: 可能是网络问题、CORS问题或服务器未启动
建议操作:
- 检查后端服务是否在 http://localhost:8000 启动
- 查看后端控制台是否有错误信息
```

#### CORS错误
```
错误类别: API
错误消息: HTTP 状态码
详细信息: Access-Control-Allow-Origin 缺失
建议操作:
- 检查后端是否配置CORS
- 确认前端和后端域名/端口一致
```

#### 数据库连接问题
```
错误类别: API
错误消息: 数据库连接失败
建议操作:
- 检查PostgreSQL服务是否运行
- 验证数据库密码和连接配置
- 查看后端日志获取详细错误
```

#### API密钥问题
```
错误类别: API
错误消息: 认证失败
建议操作:
- 检查模型配置的API密钥
- 确认API密钥是否有效且未过期
```

### 3. 查看详细错误

点击日志条目的"详情"按钮查看：
- 完整的请求/响应数据
- 错误堆栈跟踪
- 网络请求详情

## 日志级别说明

- **ERROR**: 系统错误，需要立即关注
- **WARN**: 警告信息，可能影响系统正常运行
- **INFO**: 一般信息，记录重要操作
- **DEBUG**: 调试信息，仅在开发时使用

## 日志类别说明

- **API**: API请求/响应日志
- **NETWORK**: 网络相关错误
- **ERROR**: 系统未捕获错误
- **USER**: 用户操作日志

## 最佳实践

1. **日常监控**: 定期查看日志页面，了解系统运行状态
2. **错误排查**: 遇到问题时，首先查看日志获取详细信息
3. **定期清理**: 当日志数量超过1000条时，系统会自动清理旧的日志
4. **导出备份**: 重要问题排查时，导出日志保存证据
5. **日志分析**: 使用搜索和筛选功能快速定位相关日志

## 常见问题

### Q: 日志会保留多久？
A: 日志存储在浏览器localStorage中，最多保留1000条，最新的日志会覆盖最旧的日志。清除浏览器数据会删除日志。

### Q: 刷新页面后日志会丢失吗？
A: 不会。日志存储在localStorage中，刷新页面后仍然保留。

### Q: 如何彻底清除日志？
A: 在日志页面点击"清空日志"按钮，或在浏览器开发者工具中手动清除localStorage。

### Q: 导出日志有什么格式？
A: 导出为JSON格式，包含完整的日志信息，方便程序化分析。

### Q: 日志会影响系统性能吗？
A: 不会。日志记录是异步的，并且有数量限制（最多1000条），对系统性能影响微乎其微。

## 技术细节

### 日志记录流程

1. **请求发起**: API拦截器记录请求详情
2. **响应接收**: API拦截器记录响应数据
3. **错误捕获**: 统一错误处理，记录错误详情
4. **本地存储**: 日志写入localStorage
5. **控制台输出**: 同时输出到浏览器控制台

### 日志数据结构

```typescript
interface LogEntry {
  timestamp: string;     // ISO格式时间戳
  level: 'ERROR' | 'WARN' | 'INFO' | 'DEBUG';
  category: 'API' | 'NETWORK' | 'ERROR' | 'USER';
  message: string;       // 日志消息
  details?: string;      // JSON格式的详细信息
  stack?: string;        // 错误堆栈（仅ERROR级别）
}
```
